# Intent Drift — Ubiquitous Language

## Project
A GitHub repository linked to the portal via a user's GitHub OAuth connection. One-to-one mapping with a GitHub repo. The term "project" and "repository" are interchangeable in this domain.

## Analysis
A run of the full AI pipeline (reverse spec → gap analysis → question generation) triggered on a PR. Only one Analysis can be active per PR at a time. Re-triggering replaces the previous Analysis and resets all answers.

**States:** `analyzing → ready → completed | failed`

**Completed:** An Analysis moves to `completed` only when the Respondent explicitly submits after answering every Question. Partial submissions are not allowed — the submit action is gated on all Questions having answers.

**Stale:** An Analysis in `ready` or `completed` state is marked stale when new commits are pushed to the PR after the Analysis ran. A stale warning is shown in the UI; re-triggering is always manual.

**Concurrency:** First trigger wins — if an Analysis is already in `analyzing` state for a PR, subsequent trigger attempts are rejected. The PR is unlocked when the Analysis reaches `ready`, `completed`, or `failed`.

## Spec (Original Specification)
The human-authored document describing intended behaviour for the code in a PR. Provided by the user at trigger time via paste or file upload in the portal UI. Not pulled from any external system.

## Code Input
The full file contents of all files changed in a PR (not a diff, not the entire repo). Fetched from GitHub at trigger time.

## Trigger
The act of initiating an Analysis on a PR. Any user with GitHub repository access can trigger. The person who triggers becomes the Respondent for that Analysis.

## Respondent
The single person responsible for answering all questions generated by an Analysis. Always the person who triggered the Analysis. No other user can submit answers for that Analysis.

## Gap
A divergence identified by the AI between the Spec and the reverse spec — classified by type (missing feature, deviation, undocumented addition) and severity.

## Question
An AI-generated, targeted prompt attached to a Gap, designed for the Respondent to answer. Each Gap has one or more Questions.

## AI Pipeline
Three sequential LLM calls to Anthropic Claude running server-side on the Node.js backend: reverse spec generation → gap analysis → question generation. Triggered asynchronously — the frontend polls for status using the Analysis ID.

## Decision Record
The output of a completed Analysis — a structured document capturing all Gaps, their Questions, and the Respondent's answers. Visible to all repo members in the portal and downloadable as Markdown.
